4. Data Security and Encryption

4.1 Encryption Standards

We implement industry-standard encryption to protect your data:

  • AES-256-GCM: Military-grade authenticated encryption for all sensitive data at rest
  • PBKDF2: Key derivation algorithm with 100,000 iterations (NIST-recommended) for secure key generation
    • TLS/HTTPS: Transport Layer Security for all data in transit
  • Authenticated Encryption: Prevents tampering and ensures data integrity through authentication tags

4.2 What We Encrypt

Database Fields (Automatic Encryption):

  • Stripe customer IDs (EncryptedCharField)
  • Stripe subscription IDs (EncryptedCharField)
  • Stripe payment intent IDs (EncryptedCharField)
  • All sensitive PII stored in encrypted fields

Uploaded Files (Configurable Encryption):

  • Dataset files can be encrypted when uploaded (controlled by ENCRYPT_DATASETS setting)
  • User-specific encryption keys for file-level encryption
  • Automatic encryption/decryption when files are accessed

4.3 Security Measures

  • Password Security: Passwords are hashed using Django's PBKDF2 password hasher with SHA-256
  • Access Controls: Role-based access control and authentication requirements
  • Admin Activity Logging: All administrative actions are logged for audit trails
  • Regular Security Audits: We conduct regular vulnerability assessments and security reviews
  • Secure Infrastructure: Our servers are secured with firewalls, intrusion detection, and monitoring
  • Data Backup: Regular encrypted backups of all data
  • Incident Response: Procedures in place for security incident detection and response

4.4 Key Management

  • Encryption keys are stored separately from encrypted data
  • Master encryption keys are generated using cryptographically secure random number generators
  • User-specific keys are derived using PBKDF2 with unique salts
  • Keys are never transmitted or stored in plaintext
